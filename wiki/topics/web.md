# Web

Web is the application-testing branch of the wiki. It covers how browser-facing and API-facing systems are examined, how requests and responses are interpreted, and how application logic turns into security findings.

## How to use this category
Use Web when the main object of study is a web application, API, identity flow, session model, or browser-mediated trust boundary.

## What belongs here
- intercepting proxies and traffic inspection
- parameter discovery and request shaping
- authentication, session, and authorization testing
- frameworks and tooling for application-layer assessment

## Related categories
- [[topics/recon]] — Discovery, mapping, enumeration, and target understanding before deeper validation or exploitation.
- [[topics/exploit]] — Controlled validation of weaknesses through payloads, exploit frameworks, and exploit-adjacent tooling.
- [[topics/creds]] — Credential capture, relay, cracking, token handling, and identity material as an operational category.
