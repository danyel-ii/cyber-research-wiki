# Kali Linux

## Overview
Kali Linux is a security-focused Linux distribution presented by its maintainers as a platform for penetration testing and related authorized security work. In this wiki it is tracked as an assessment environment and product ecosystem, not as a substitute for methodology, scope control, or authorization.

## What the current source set supports
- Kali is positioned as a purpose-built penetration-testing distribution for security professionals.
- The project offers multiple deployment forms including installer images, live media, virtual machines, containers, cloud images, WSL packages, ARM builds, and NetHunter-related mobile variants.
- Tool scope is managed in part through metapackages that allow narrower or broader install profiles.
- Custom image generation is a first-class workflow, with public build scripts and documentation for rebuilding or tailoring official images.

## Research notes
- The corpus strongly supports Kali as a flexible delivery platform for assessment tooling.
- The corpus does not by itself justify vendor superlatives such as "industry standard"; those should be treated as project positioning.
- The corpus mixes project documentation and OffSec branding, so organizational relationship claims should remain carefully scoped until further primary governance sources are ingested.

## Assessment implications
- Kali can reduce environment setup time for authorized tests.
- The availability of many form factors can help align an environment to a target domain, operator constraints, or lab setup.
- Ease of access to tools does not remove the need for explicit rules of engagement, logging discipline, and minimal-scope tool selection.
