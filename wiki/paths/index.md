# Curriculum Paths

## Overview
This section turns the wiki into a learnable system rather than a loose collection of notes. Each path is an ordered reading track with prerequisites, outcomes, and explicit extension points so the curriculum can grow as the wiki grows.

## How to use this section
- Start with one path rather than hopping randomly across the wiki.
- Read the pages in order unless a path marks an item as optional.
- Use source pages to verify claims, not as the first stop when learning a topic.
- Return to this hub after finishing a path and choose the next branch.

## Path design rules
- Paths should be stable even when individual pages change.
- Paths should prefer concept pages before source pages.
- Every path should state what new pages would cause it to expand.
- Every path should have a "next when this grows" section so maintainers know where to attach new material.

## Curriculum maintenance model
When new material is ingested, update the curriculum in this order:
1. Decide whether the new page belongs inside an existing path.
2. Add it to the smallest sensible path rather than many paths at once.
3. If repeated additions cluster around one area, split out a new path.
4. Update this hub when a path is created, renamed, or changes level.
5. Record the change in [[log]] when path structure materially changes.
